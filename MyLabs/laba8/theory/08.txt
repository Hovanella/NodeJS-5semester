1. Поясните назначение заголовка Content-Type.
    Ответ : Content-Type - заголовок, который указывает тип данных, которые содержатся в теле сообщения. Например, если в теле сообщения содержится HTML-документ, то заголовок будет выглядеть так: Content-Type: text/html.

2.  Поясните назначение заголовка Accept.
    Ответ : Accept - заголовок, который указывает типы данных, которые клиент может обработать. Например, если клиент может обработать только HTML-документы, то заголовок будет выглядеть так: Accept: text/html.

3.Для чего используется значение Multipart/form-data заголовка Content-Type?
    Ответ : Значение Multipart/form-data заголовка Content-Type используется для передачи данных формы, которые содержат файлы.

4.Как с помощью   тега form, обеспечить значение Multipart/form-data
заголовка Content-Type.
    Ответ : Для того, чтобы обеспечить значение Multipart/form-data заголовка Content-Type, необходимо в теге form указать атрибут enctype="multipart/form-data".

5.Какое значение заголовка Content-Type отправляется тегом form в запросе по умолчанию.
    Ответ : Значение заголовка Content-Type по умолчанию - application/x-www-form-urlencoded. Это значение указывается в теге form, если не указан атрибут enctype.

6. Где и в каком формате передаются параметры в GET-запросе?
Ответ: параметры передаются в строке запроса, в виде пары имя=значение, разделенных символом &. Например, http://www.example.com/index.php?name=John&age=25. В этом случае параметры name и age передаются в GET-запросе.

7. Где и в каком формате передаются параметры в POST-запросе?
Ответ: параметры передаются в теле запроса, в виде пары имя=значение, разделенных символом &. Например, name=John&age=25. В этом случае параметры name и age передаются в POST-запросе.

8.Поясните понятие JSON?
Ответ: JSON (JavaScript Object Notation) - это текстовый формат обмена данными, основанный на синтаксисе языка JavaScript. JSON представляет собой набор пар имя-значение, разделенных запятыми. В качестве значений могут быть использованы строки, числа, логические значения, массивы, объекты и null. Все значения должны быть заключены в двойные кавычки. Например, {"name":"John","age":25}.

9. Поясните понятие XML?
Ответ: XML - это язык разметки, который используется для описания структуры данных. XML-документ представляет собой набор элементов, которые могут содержать другие элементы. Все элементы должны быть заключены в угловые скобки. Например, <name>John</name><age>25</age>.
